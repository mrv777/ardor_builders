CryptoPay
CryptoPay is a payment portal that can be plugged into any e-commerce site using API keys generated by the sellers when they sign up. CryptoPay provides simple APIs (just like Stripe) for the sellers to request transactions from the users by just the user's email and the amount to be transacted. This would allow e-commerce sellers to accept payments in Ardor and Ignis from their customers.

The customers will get a notification on their mobile app when they check-out a cart on such e-commerce sites if they have used the same email that they use for logging into the CryptoPay mobile app in order to sign up on the e-commerce site where they have checked out a transaction. If the users have used a different email to sign up into the e-commerce site, they will get a simple transaction code to enter on their mobile app to confirm their order. The mobile app would require the buyers to scan the QR code of their passphrase in order to confirm their transaction with the e-commerce site where they have made the purchase from.
Ardor.gifts
Ardor Gifts is a paper wallet creator for Ardor with the intent to connect the user into the experience of gifting ARDR wallets in real life e.g. as birthday present or Christmas gift. Also Ardor Gifts enables users to safely create their Ardor wallet offline from Windows, Linux or Mac in air gapped environment.

Ardor Gifts has been developed with an extendible way mentality, in order to enables users to submit new designs for their wallets. We will release a "build-your-design" starter pack in the following days.

The aim is to bring tools and contracts with some real UX/UI experience for the users in order to make Ardor more desirable for the average Joe.

Feel free to contact me on slack @Tom Yum for any ideas!
Tickets and Memberships
We use account properties to sell tickets/different memberships to Wordpress websites, and scan users' accounts for them when they want to access private sections of the site. We use the IGNIS currency and blockchain, and the SIGBRO app to make it easier for the ticket issuer and the end-user.

Features: User sends 1-9 IGNIS and receives a ticket/membership (Standard member)
or 10-49 IGNIS to get a Silver membership (so he gets access to more private pages).
When the user logs in to a website our script checks his account for the ticket/membership
- if the user has a valid ticket/membership, he gets access
- if the user does not have a valid ticket/membership, access is denied!
Private pages are defined by simply pasting the relevant shortcode anywhere on the page.
Outgoing transactions (when setting the account property and value) are signed locally.

The script can be quite easily changed so it checks for, for example, a specific asset instead of account properties + values.
